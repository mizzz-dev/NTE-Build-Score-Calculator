# AGENTS.md

## 目的
本リポジトリでCodexが実装を行う際のルールを定義する。

## 開発原則
1. ウォーターフォール型で進める。
2. 要件定義・設計が未確定の実装は禁止。
3. PR本文・コメント・ドキュメントは日本語で統一する。
4. 変更はIssueまたは設計根拠に紐づける。

## 実装ルール
- 無関係なリファクタをしない。
- 不要な依存追加をしない。
- 破壊的変更時は移行手順を明記する。
- スコア計算ロジックは設定駆動を優先する。

## ドキュメントルール
- 要件: `docs/requirements.md`
- 画面設計: `docs/ui-design.md`
- スコア仕様: `docs/scoring-spec.md`
- データ設計: `docs/data-design.md`
- 管理要件: `docs/admin-requirements.md`
- ロードマップ: `docs/roadmap.md`
- リリース確認: `docs/release-checklist.md`

## 法務・権利
- 非公式ファンツールであることを明記する。
- 公式素材利用は規約確認完了まで「要確認」扱いにする。

## 品質ルール
- 実装PRにはテスト結果を記載する。
- 仕様未確定事項は「要確認」を明記し、仮実装しない。
