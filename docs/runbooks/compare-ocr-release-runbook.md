# `/compare` OCR 正式展開 Runbook（Issue #125）

- 作成日: 2026-05-15
- 対象Issue: #125
- 関連: #113, #115, #117, #119, #121, #123
- 対象: `/compare` OCR入力補助の正式展開後運用
- 前提: OCRアルゴリズム、DB、auth、infra、deployment、保存payload仕様、ランキング仕様は変更しない
- 方針: 本Repositoryは非公式ファンツール。公式素材利用可否は継続して「要確認」

## 1. 正式展開判定サマリ
- 判定種別: **条件付き正式展開可**
- 判定理由（要約）:
  1. 第1〜第3サイクルで重点KPI（低性能端末p95 / fallback率 / 比較前確認離脱率）が連続改善。
  2. A/B取り違え誤反映率は3サイクル連続で0.0%。
  3. 保存payload・共有URL・ランキングpayload互換を維持（OCRメタ非混入）。
  4. `image_quality_low` / `unresolved_items_remaining` 起因の運用リスクは残存するため、監視条件付きで正式展開。

## 2. 正式展開後の監視KPI
| KPI | 通常監視しきい値 | 警戒しきい値（即時運用確認） | rollback判定しきい値 |
|---|---:|---:|---:|
| OCR処理時間 p95（全体） | <= 3.9秒 | > 3.9秒 が2営業日連続 | > 4.2秒 が2営業日連続 |
| OCR処理時間 p95（低性能端末） | <= 4.0秒 | > 4.0秒 が2営業日連続 | > 4.3秒 が2営業日連続 |
| A/B取り違え誤反映率 | 0.0% | > 0.0% を1件でも検知 | > 0.0% を1件でも検知（重大） |
| 未確定項目残存率（比較直前） | <= 13.0% | > 13.0% が2営業日連続 | > 16.0% が2営業日連続 |
| 比較前確認離脱率 | <= 4.0% | > 4.0% が2営業日連続 | > 5.0% が2営業日連続 |
| fallback率 | <= 8.5% | > 8.5% が2営業日連続 | > 10.0% が2営業日連続 |
| 手動補正率 | <= 18.0% | > 18.0% が2営業日連続 | > 21.0% が2営業日連続 |

- A/B取り違え誤反映は安全性上の重大項目として「1件検知で即時停止検討」。
- しきい値は第1〜第3サイクル実績（4.2→4.0→3.8秒等）を基準に設定。

## 3. しきい値超過時の対応方針
### 3.1 警戒しきい値超過（rollback未満）
1. 当日中に運用ログへ記録（日時、対象KPI、端末カテゴリ、要因内訳）。
2. 翌営業日までに暫定対処方針を記載（文言調整可否、導線見直し要否）。
3. 2営業日連続超過で改善Issue起票（ラベル: `ocr`, `compare`, `ops`）。
4. 互換性回帰（保存payload / 共有URL / ランキングpayload）を最小再確認。

### 3.2 rollback判定しきい値超過
1. 直ちに`/compare` OCR導線をfeature flagで無効化。
2. 影響時間帯・症状・対象端末カテゴリを運用ログへ追記。
3. 手動入力導線の正常性確認後、限定導入モードへ戻すかを運用レビューで判断。
4. 再開前に最低回帰確認を再実施（第7章）。

## 4. rollback条件
1. A/B取り違え誤反映率が0.0%を逸脱（1件でも発生）。
2. 未確認OCR値が比較計算へ混入した事象を1件でも検知。
3. 保存payload / 共有URL / ランキングpayloadの互換性破壊を検知。
4. 低性能端末p95 > 4.3秒 が2営業日連続。
5. fallback率 > 10.0% が2営業日連続。
6. 比較前確認離脱率 > 5.0% が2営業日連続。

## 5. rollback手順
1. feature flagで`/compare` OCR導線を無効化する。
2. `/compare` のUIでOCR起動導線が表示されないことを確認する。
3. OCR未使用時の手動入力のみで比較結果算出可能であることを確認する。
4. KPI計測を一時停止し、停止理由・開始日時・暫定再開条件を運用ログに記録する。
5. 保存payload・共有URL・ランキングpayload互換の最低回帰確認を実施する。
6. 影響範囲と暫定恒久対応（改善Issue）を `docs/logs/` に追記する。

## 6. `/compare` OCR導線を無効化する場合の確認手順
1. `/compare` でOCR導線ボタンが非表示であること。
2. A/Bの手動入力導線が両方有効であること。
3. 比較前確認ガードが手動入力のみでも期待通り動作すること。
4. OCR関連メッセージ（エラー/待機）が表示されないこと。
5. 比較結果の算出値がOCR無効化前の手動入力結果と一致すること（代表ケース）。

## 7. 最低回帰確認手順（互換性）
### 7.1 保存payload互換
1. OCR未使用で比較結果を保存。
2. payloadに `confidence` `matchType` `raw text` `candidates` が含まれないことを確認。

### 7.2 共有URL互換
1. OCR無効化状態で共有URLを生成。
2. 既存キー構造・エンコード仕様と差分がないことを確認。

### 7.3 ランキングpayload互換
1. OCR無効化状態でランキング連携対象payloadを確認。
2. フィールド構造・型・必須項目が変更されていないことを確認。

## 8. ADR要否
- 結論: **ADR新規作成は不要**
- 理由:
  1. アーキテクチャ変更を伴わず、既存方針を運用面で確定したのみ。
  2. 主対象は運用判定基準とRunbookであり、設計原則の恒久変更ではない。
  3. 判断根拠と運用条件は `docs/logs/compare-ocr-release-decision.md` で追跡可能。
