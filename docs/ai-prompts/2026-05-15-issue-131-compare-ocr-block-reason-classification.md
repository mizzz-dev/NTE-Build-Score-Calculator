# AIプロンプトログ: Issue #131 `/compare` 比較ブロック理由の分類とOCRガイド出し分け

- 日付: 2026-05-15
- 対象Issue: #131
- 目的: 入力エラーのみの比較ブロック時にOCR未確定向けガイドを表示しないようにし、OCR由来のブロック理由がある場合だけ未確定残件数と確認順ガイドを表示する。

## 指示要約
1. `docs/ai-protocol/PROMPT.txt` を最優先で遵守。
2. 比較ブロック理由を入力エラー系とOCR確認系に分類。
3. OCR由来理由がある場合のみ未確定残件数と確認順ガイドを表示。
4. 未確定0件まで比較ブロックする既存ガードを維持。
5. OCRアルゴリズム・OCR信頼度計算・payload仕様等の禁止領域は変更しない。
6. `docs/current-status.md` `docs/active-issues.md` `docs/logs` `docs/ai-prompts` を更新。

## 実施結果サマリ
- ガード関数の返却へ理由カテゴリ情報（input/ocr）とOCR理由有無フラグを追加。
- UIはOCR理由有無フラグに応じてOCRガイド表示を制御。
- 入力エラーのみケースとOCR理由含有ケースの単体テストを追加。
