# Issue #97 用AIプロンプト（/score OCR処理時間 p95 未達改善）

## 目的
PR #96 の第2サイクルログで唯一未達となった OCR処理時間 p95（4.1秒、目標4.0秒以内）を、`/score` OCR入力補助のUX改善で改善する。特に低スペック端末カテゴリ p95=4.9秒への対応を優先する。

## 前提
- 対象は `/score` OCR入力補助のみ。
- OCRアルゴリズムの大規模変更は行わない。
- 低信頼度候補の自動確定は禁止を維持する。
- OCR結果や画像データを保存payload・ランキングpayload・共有URLへ含めない。
- `/card` `/compare` へのOCR展開は保留する。

## 実施項目
1. 第2サイクルログの未達KPIと原因仮説を整理する。
2. 低スペック端末向けに、待機中メッセージを経過時間に応じて段階化する。
3. 長時間処理時の「継続待機 / 別画像で再試行 / 手動テキストfallback / 手動入力」判断を明示する。
4. 画像サイズ・低解像度・ぼやけ画像に関する事前ガイドを最小追加する。
5. 関連テストを追加し、文言回帰を防止する。
6. `docs/current-status.md` `docs/active-issues.md` を更新する。
7. 作業ログを `docs/logs/` に保存する。

## 完了条件
- p95未達の原因仮説と対策がRepository内に記録される。
- 低スペック端末向け待機/代替導線が改善される。
- OCR非混入方針と `/score` 限定適用が維持される。
- `pnpm lint` `pnpm test` `pnpm build` が成功する。
