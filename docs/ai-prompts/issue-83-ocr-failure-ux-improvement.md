# Issue #83 用AIプロンプト（/score OCR失敗時UX改善）

## 目的
`/score` OCR入力補助で、OCR読み取り失敗・マッピング失敗・低信頼度候補・端末性能遅延が発生しても、ユーザーが次アクション（再試行/手動テキストfallback/手動入力）を理解して完了まで進めるUXを実装する。

## 制約
- OCRアルゴリズムは変更しない。
- `/card` `/compare` へは影響させない。
- 保存payload/ランキングpayload/共有URLへOCR結果や画像データを含めない。
- 外部OCR API連携・画像サーバー保存・DB migration追加を行わない。

## 実装観点
1. 失敗時エラー文言を行動可能な日本語へ改善する。
2. 再試行導線（同一画像/別画像/手動テキストfallback/手動入力）を明示する。
3. 低信頼度・未解決項目を可視化し、補正完了条件を表示する。
4. フォーム反映可否と理由を表示する。
5. 関連テスト・運用ドキュメント・作業ログを更新する。
