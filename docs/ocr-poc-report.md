# OCR PoC検証レポート（Issue #67）

## 1. 概要
- 目的: `docs/ocr-requirements.md` の方針に基づき、`/score` 向け入力補助としてブラウザ内OCRの実装可否を検証する。
- 範囲: PoCユーティリティ実装と評価レポートのみ。
- 非範囲: `/score` 本番UI接続、`/card`・`/compare` 展開、外部OCR API連携、保存payload変更、DB変更。

## 2. 検証方針
### 2.1 OCR候補
1. Tesseract.js
2. OCRAD.js

### 2.2 比較観点
- 読み取り精度（ステータス名・数値・`%`・小数点）
- 処理時間（端末性能差を含む）
- バンドル影響
- セキュリティ/プライバシー整合
- 手動補正前提の運用妥当性

## 3. PoC最小実装
### 3.1 実装内容
- OCR生テキスト行から `OcrEquipmentDraft` を生成する最小パーサーを追加。
- 公開マスタステータスへのマッピング関数を追加。
- 未一致項目を `unresolvedReason` で明示し、手動補正前提を保証。
- OCR候補比較関数を追加し、採用判定をコード上で再利用可能化。

### 3.2 データ境界
- 画像データはPoC内で扱わず、ブラウザ内OCRエンジンの出力文字列だけを入力にする。
- OCR結果は `OcrEquipmentDraft` の一時状態に限定する。
- 既存の保存payload・ランキングpayload生成ロジックには接続しない。

## 4. 画像条件（仮設定）
- 形式: `image/png`, `image/jpeg`, `image/webp`
- 最大サイズ: 5MB
- 推奨解像度: 短辺1080px以上
- 注意: 短辺720px未満は精度低下警告を出す想定

## 5. 読み取り対象
- ステータス名
- 数値
- `%`
- 小数点

※ 装備/キャラクター等の拡張は次PR以降で要確認。

## 6. セキュリティ/プライバシー
- サーバー送信・永続保存なしのブラウザ内処理を前提とする。
- OCR結果は画面内ドラフトのみで保持し、保存payloadへ直接含めない。
- OCR低信頼度・未一致は必ず手動補正する。

## 7. 検証結果
### 7.1 候補比較
| 候補 | 採用判定 | 理由 |
|---|---|---|
| Tesseract.js | 条件付き採用 | ブラウザ完結で方針整合。バンドル増・端末差が大きいため遅延ロード前提。 |
| OCRAD.js | 保留 | 軽量性は利点だが、`%`/小数点/多言語ステータス名の精度懸念が残る。 |

### 7.2 マッピング課題
- 表記ゆれ（例: `Crit Rate`, `会心率`, `치명타 확률`）は別名辞書の継続拡充が必要。
- 公開マスタ未登録語は自動確定せず、候補表示+手動選択にフォールバックする。

### 7.3 処理時間・端末性能差
- PoC時点は実測ベンチマーク未実施（要確認）。
- 次段で低スペック端末のタイムアウト閾値（例: 3〜5秒）を検証する。

### 7.4 バンドル影響
- Tesseract.jsはバンドル増加が想定されるため、`/score`画面内でもOCR起動時の遅延読込を必須条件とする。

## 8. 採用可否（Issue #67 判定）
- **結論: 条件付き採用**
- 条件:
  1. `/score` OCR補助UIへ限定導入
  2. OCRエンジンは動的importで遅延読込
  3. 低信頼度・未一致は自動反映せず手動補正必須
  4. 保存payload・ランキングpayloadへの直接混入禁止を維持

## 9. 次PRへの入力
- `/score` 向けOCR入力補助UI（画像選択・注意表示・候補反映）を追加する。
- Tesseract.js実測（主要ブラウザ/端末）を実施し、閾値とフォールバック導線を確定する。
- 公開マスタ同義語辞書の運用手順を定義する（管理画面連携は要確認）。

## 10. 要確認
- 韓国語フォントでの誤認識率
- 端末メモリ不足時の失敗ハンドリング
- 利用規約上の画像取り扱い注意文言（継続して「要確認」）

## 11. Issue #69 実装差分（最小追記）
- `/score` にOCR入力補助UI（画像選択・一時ドラフト表示・手動補正前提）を追加。
- 画像はブラウザ内一時処理のみとし、保存payload/ランキングpayload/共有URLにはOCR情報を含めない。
- 初期実装はPR #68の「OCR生テキスト行→ドラフト生成PoC」をUI接続し、低信頼度・未解決項目は自動反映を禁止。

## 12. Issue #71 実装差分（OCRアダプタ/補正UI強化）
- `src/features/ocr/lib/adapter.ts` にOCR実行アダプタ層を追加し、`File`受け取り→OCR生テキスト行を返すインターフェースを分離。
- 初期実装は安全なbrowser stubとし、画像をブラウザ内で読み込むのみでサーバー送信・外部OCR API連携は実施しない。
- `/score` OCR補助UIをアダプタ呼び出しに接続。stub失敗時は従来の「OCR生テキスト貼り付け」からドラフトを再生成する導線を維持。
- OCR状態を `processing/success/error/needs_review` で扱い、手動補正待ちを明示。
- 一時ドラフト編集UIを強化し、装備タイプ・メインキー/値・サブキー/値をユーザーが明示的に修正可能化。
- 公開マスタ由来のステータス候補（`statuses[].code`）を優先して選択肢表示し、候補不足時のみ既定キーへフォールバック。
- 低信頼度/未解決でも、ユーザーが手動補正した項目は `requiresManual=false` としてフォーム反映可能にした。
- OCR結果/画像データを保存payload・ランキングpayload・共有URLへ含めない既存方針を維持（保存・共有関連ロジックの仕様変更なし）。
