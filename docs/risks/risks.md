# リスク一覧

最終更新: 2026-05-12

## 1. OCR精度リスク
- 内容: 文字認識誤りにより誤ったステータス候補が提示される。
- 影響: 誤入力のまま確定するとスコア評価が不正確になる。
- 対策: 低信頼度項目の警告表示、手動補正必須、候補提示改善、公開マスタalias強化。

## 2. payload混入リスク
- 内容: OCRメタ情報（confidence, raw text等）が保存payloadへ混入する。
- 影響: 保存/共有/ランキング互換崩壊、後方互換性問題。
- 対策: OCRメタ情報は画面内一時状態のみに限定し、payloadBuilder入力契約を固定。

## 3. bundle size / 端末性能リスク
- 内容: OCRライブラリ導入で初期ロードや実行時間が増大。
- 影響: 低スペック端末でUX悪化、離脱率増加。
- 対策: Tesseract.js遅延読み込み、処理中UI明示、失敗時の手動入力fallback、計測運用。

## 4. 公開マスタ誤投入リスク
- 内容: ステータス名/aliasの誤登録によりOCRマッピングが誤誘導される。
- 影響: 誤候補提示増加、補正負荷増、計算結果の信頼性低下。
- 対策: マスタ変更レビュー手順、差分検証、主要ステータス回帰テスト、投入前チェックリスト整備。

## 5. OCR品質評価運用不足リスク
- 内容: 指標定義や継続計測が不十分で、精度劣化・端末差・fallback有効性を検知できない。
- 影響: `/card` `/compare` 展開判断を誤り、不安定品質のまま適用範囲が拡大する。
- 対策: `docs/reports/ocr-quality-evaluation.md` のKPIを週次運用し、未達時は失敗分類単位で改善Issueを起票する。
