# 画面・デザイン方針

## 1. デザインコンセプト
- ダーク・ネオン・近未来都市
- 情報密度は高く、操作はシンプル
- スコアの可読性を最優先

## 2. 画面一覧（初期リリース）
- トップページ
- スコア計算ページ
- ビルド比較ページ
- ビルドカード生成ページ
- プリセット一覧ページ
- ステータス一覧ページ
- 使い方ページ
- FAQページ
- アップデート情報ページ
- お問い合わせ / フィードバックページ

## 3. レイアウト方針
- スマホ: 1カラム
- PC: 入力/結果の2カラム
- ナビゲーション: PCはサイドバー、スマホはハンバーガーメニュー
- トップページはLP形式とし、ツールへの即時遷移導線を配置

## 4. UIコンポーネント方針
- 入力フォームはステータス単位で統一コンポーネント化
- スコア表示はランク色分け（SSが最大、10段階程度）
- 比較画面は差分強調表示
- カード生成はプレビュー即時反映

## 5. 多言語表示方針
- 表示名称は `日本語（英語）` 基本で管理し、韓国語対応を追加
- 文言キーで管理し、ハードコードを避ける

## 6. アクセシビリティ方針
- キーボード操作の主要導線確保
- コントラスト確保
- ラベルと入力欄の明確な紐付け
