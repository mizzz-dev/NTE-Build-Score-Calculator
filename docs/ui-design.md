# 画面・デザイン方針

## 1. 本ドキュメントの位置づけ
- 本書は「実装前の基本設計（ルーティング・画面遷移・共通レイアウト）」を定義する。
- 対象はユーザー向け公開画面（管理画面は別途 `docs/admin-requirements.md` / 今後の管理画面設計で定義）。
- 未確定仕様に依存する機能は「要確認」とし、仮実装前提の記載は行わない。

## 2. デザインコンセプト
- ダーク・ネオン・近未来都市
- 情報密度は高く、操作はシンプル
- スコアの可読性を最優先
- 非公式ファンツールであることをフッターで明示（公式素材利用は要確認）

## 4. ルーティング定義（確定版）

| 画面ID | 画面名 | URL | 目的 | 主な入力 | 主な出力 |

### 4.1 URL設計ルール
- 初期リリース時点は上記固定パスを採用し、動的URL（例: `/score/:id`）は導入しない。
- クエリパラメータは画面状態の一時表現に限定（例: フィルタ条件）。
- 共有URL仕様は詳細設計フェーズで定義（本書では導線責務のみ定義）。

## 5. 画面遷移設計

### 5.1 主要導線（LP起点）
- LPファーストビューに主要CTAを3つ配置し、以下へ直接遷移させる。
  - 「スコアを計算する」→ `/score`
  - 「ビルドを比較する」→ `/compare`
  - 「カードを作成する」→ `/card`
- LP中段には補助導線として `/presets` `/guide` `/faq` を配置する。

### 5.2 グローバル遷移
- 全画面共通ナビゲーションから、主要10画面へ常時遷移可能とする。
- スコア計算結果からの二次導線を必須化する。
  - `/score` → `/compare`（比較へ追加）
  - `/score` → `/card`（カード生成へ送る）
- FAQ / 使い方 / お問い合わせは相互リンクし、自己解決導線を確保する。

## 6. レスポンシブレイアウト方針

### 6.1 ブレークポイント方針
- スマホ: 〜767px（1カラム）
- タブレット: 768〜1023px（1〜2カラム可変）
- PC: 1024px〜（2カラム基調）

### 6.2 画面領域の基本構成
- 共通: `ヘッダー / ナビゲーション / メインコンテンツ / フッター`
- PC: 左サイドバー（ナビ） + 右メインコンテンツ
- タブレット: 上部ヘッダー + 必要時ドロワー式ナビ
- スマホ: 上部ヘッダー + ハンバーガーメニュー + 1カラム本文

### 6.3 スコア計算画面のレイアウト
- PC: 入力パネル（左）/ 結果パネル（右）の2カラム固定。
- タブレット: 縦積みを基本とし、横幅確保時のみ2カラム。
- スマホ: 入力→結果の縦順。主要結果（総合スコア）は画面上部に固定表示（スクロール追従は詳細設計で判断）。

## 7. ナビゲーション挙動定義

### 7.1 PCサイドバー
- 既定は展開状態（ラベル表示あり）。
- 折りたたみ操作を許可し、折りたたみ時はアイコン表示のみ。
- 現在ページをハイライト表示し、階層を持たない単層ナビとする。

### 7.2 スマホハンバーガーメニュー
- ヘッダー左上に配置。
- タップで全画面ドロワーを開閉。
- メニュー表示中は背景スクロールをロック。
- 画面遷移後は自動で閉じる。
- 「主要機能（計算/比較/カード）」を上部固定表示し、その他を下段表示。

## 8. 共通UI責務（ヘッダー / フッター / ナビ）

### 8.1 共通ヘッダー
- 責務: ブランド表示、現在地補助、主要操作入口（メニュー/言語切替予定領域）
- 非責務: 画面固有の入力フォーム配置

### 8.2 共通フッター
- 責務: 非公式ファンツール表記、著作権・権利表記、補助リンク（FAQ/問い合わせ/更新情報）
- 公式素材利用に関する文言は「要確認」を維持

### 8.3 共通ナビゲーション
- 責務: 主要10画面への遷移、現在地表示、モバイル時の開閉制御
- 非責務: 画面内セクションの細粒度タブ制御

## 9. 主要画面の実装可能粒度定義

### 9.1 スコア計算（`/score`）
- 必須セクション: 条件入力 / ステータス入力 / 計算結果 / 関連導線
- 最低限の出力: 総合スコア、評価ランク、主要内訳
- 要確認: 厳密計算式、キャラ別補正、セット効果算入

### 9.2 ビルド比較（`/compare`）
- 必須セクション: 比較対象選択 / 比較結果表 / 差分ハイライト
- 最低限の出力: 各指標の差分、優勢表示

### 9.3 ビルドカード生成（`/card`）
- 必須セクション: 元ビルド選択 / デザイン選択 / プレビュー / 出力操作
- 最低限の出力: プレビュー画像、保存導線、共有導線
