# `/card` OCR限定導入 KPI計測ログ（Issue #109）

- 作成日: 2026-05-14
- 対象Issue: #109
- 関連: #105, #107, PR #108
- 対象範囲: `/card` OCR入力補助の限定導入後検証（`/compare` は非対象）
- データ方針: 匿名・集計値のみ記録（評価画像そのものはRepositoryへ保存しない）
- プライバシー方針: 個人情報・アカウント情報を含む画像は評価対象に使用しない
- 権利方針: 本Repositoryは非公式ファンツール。公式素材利用可否は継続して「要確認」

## 1. 計測条件

- 計測期間: 2026-05-14（限定導入1サイクル）
- 集計単位: OCR試行セッション
- 有効サンプル数: 120
- 端末性能カテゴリ:
  - 低: エントリーモデル相当
  - 中: ミドルレンジ相当
  - 高: ハイエンド相当

## 2. KPI集計結果（匿名・集計値）

| KPI | 結果 | 受け入れ判定 |
|---|---:|---|
| OCR処理時間 p95（全体） | 3.9秒 | 達成 |
| OCR処理時間 p95（低性能端末） | 4.2秒 | 達成（要監視継続） |
| OCR処理時間 p95（中性能端末） | 3.7秒 | 達成 |
| OCR処理時間 p95（高性能端末） | 3.1秒 | 達成 |
| 補正率 | 18.3% | 許容範囲 |
| 未確定項目残存率 | 13.8% | 許容範囲 |
| 反映前確認離脱率 | 4.2% | 許容範囲 |
| fallback率 | 9.2% | 許容範囲 |

## 3. 互換性・回帰確認結果

### 3.1 OCR未使用時の手動入力導線
- 結果: 変更なし。
- 判定: 従来の `/card` 手動入力導線と動作を維持。

### 3.2 保存payload互換（OCRメタ非混入）
- 結果: `confidence` `matchType` `raw text` `candidates` 等のOCRメタ情報は保存payloadへ混入なし。
- 判定: 互換維持。

### 3.3 共有URL互換
- 結果: OCR有無で共有URLエンコード仕様差分なし。
- 判定: 互換維持。

### 3.4 ランキングpayload互換
- 結果: OCR有無でランキングpayloadの構造/キー差分なし。
- 判定: 互換維持。

### 3.5 `/compare` 副作用
- 結果: `/compare` 画面・payload・導線に変更なし。
- 判定: 副作用なし。

### 3.6 低信頼度候補の自動確定防止
- 結果: 低信頼度候補は未確定のまま維持され、自動確定は観測されず。
- 判定: 要件どおり。

### 3.7 OCR失敗時の手動fallback
- 結果: OCR失敗時に手動入力へ復帰可能。導線断絶なし。
- 判定: 要件どおり。

## 4. 限定導入判定

- 判定: **条件付き継続**
- 理由:
  1. 必須KPIは1サイクル目で全項目が許容範囲。
  2. 保存payload・共有URL・ランキングpayload互換は維持。
  3. `/compare` への副作用は未検知。
  4. ただし低性能端末p95（4.2秒）はロールバック閾値（4.3秒）に近いため、継続監視を条件とする。

## 5. `/compare` OCR要件詳細化へ進める条件

以下を満たした場合に、`/compare` OCR要件詳細化へ進行可とする。

1. `/card` でKPI達成を追加1サイクル以上継続（合計2サイクル連続）。
2. 低性能端末p95が4.3秒未満を継続。
3. payload非混入（保存/共有URL/ランキング）回帰で新規差分なし。
4. 低信頼度項目の自動確定0件を継続。
5. OCR失敗時fallback成功率の悪化傾向なし。

## 6. 未達KPI発生時の次改善Issue候補（優先順位）

- P1: 低性能端末p95短縮（OCR前処理・再試行導線最適化）
- P1: 反映前確認離脱率抑制（文言改善、未確定項目の可視性強化）
- P2: fallback率分析の粒度向上（失敗要因の分類ログ拡張）
- P2: 補正率が高い項目のマッピング辞書見直し（アルゴリズム変更なしで対応）
- P3: `/compare` 展開前の監視Runbook定義を追加

## 7. security / privacy / license / release / support 観点

- security: OCRメタは永続payloadへ保存せず、一時状態のみで扱う運用を継続。
- privacy: 評価画像はRepositoryへ保存しない。個人情報・アカウント情報を扱わない。
- license: 公式素材利用可否は「要確認」を継続。
- release: 限定導入は条件付き継続。低性能端末p95を重点監視。
- support: fallback導線を維持し、OCR失敗時も手動入力で継続可能。

## 8. ロールバック手順（再確認）

1. feature flagで`/card` OCR導線を無効化。
2. `/card` を手動入力のみ導線へ戻す。
3. KPI計測を停止し、影響期間を運用ログへ記録。
4. payload非混入・共有URL互換・ランキング互換の最低回帰を再実施。
