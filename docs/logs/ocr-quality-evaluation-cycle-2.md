## 5. 初回評価からの差分評価（重点観点）

### 5.1 低解像度・ぼやけ画像
- 低解像度は名称認識成功率が `91.9% → 93.6%`（+1.7pt）で改善したが、受け入れ基準95%は未達。
- ぼやけ画像は名称認識成功率が `88.9% → 91.2%`（+2.3pt）まで改善したが、補正率29.2%で依然高い。

### 5.2 韓国語表示
- 韓国語の名称認識成功率は `89.6% → 93.9%`（+4.3pt）、マッピング成功率は `91.8% → 94.6%`（+2.8pt）まで改善。
- ただし全体受け入れ水準（95%）には未達で、alias辞書運用の継続改善余地あり。

### 5.3 低スペック端末
- 全体p95は `4.3秒 → 4.1秒` に短縮したが、閾値4.0秒は未達。
- 低スペック端末単体では p95=4.9秒で、待機時間最適化が継続課題。

### 5.4 未解決候補ガイダンス（PR #88）
- 低信頼度候補の手動補正誘導率は `91.2% → 94.0%`（+2.8pt）で改善。
- 手動補正率は `23.3% → 19.8%`（-3.5pt）となり、20%以下を初回達成。
- UI誘導不足分類は `2件（3.3%）→ 1件（1.0%）` へ減少し、未解決候補ガイダンスの改善効果を確認。

## 7. `/card`・`/compare` 展開可否の判断材料
- 今回は7/8 KPIが達成し、初回より全KPIで悪化なし。
- ただし「OCR処理時間 p95 <= 4.0秒」が未達（4.1秒）であり、正式な展開判定条件（全KPI達成）を満たさない。
- 結論: `/card` `/compare` への展開は現時点で **保留継続**。次サイクルで処理時間KPI達成を確認後に再判定する。
