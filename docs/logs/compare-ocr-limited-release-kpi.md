- 対象範囲: `/compare` OCR入力補助の限定導入後KPI計測・互換性確認（追加実装は最小限）
- データ方針: 匿名・集計値のみ記録（評価画像そのものはRepositoryへ保存しない）
- プライバシー方針: 個人情報・アカウント情報を含む画像は評価対象に使用しない
- 権利方針: 本Repositoryは非公式ファンツール。公式素材利用可否は継続して「要確認」

## 1. 計測条件

- 計測期間: 2026-05-14（限定導入 第1サイクル）
- 集計単位: `/compare` OCR試行セッション
- 有効サンプル数: 128
- 端末性能カテゴリ:
  - 低: エントリーモデル相当
  - 中: ミドルレンジ相当
  - 高: ハイエンド相当

## 3. 互換性・回帰確認結果

### 3.1 OCR未使用時の手動入力導線・比較結果
- 結果: OCR未使用時は従来の手動入力導線を維持。比較結果算出ロジック差分なし。
- 判定: 互換維持。

### 3.2 保存payload互換（OCRメタ非混入）
- 結果: OCR使用時も `confidence` `matchType` `raw text` `candidates` 等のOCRメタが保存payloadへ混入しないことを確認。
- 判定: 互換維持。

### 3.3 共有URL互換
- 結果: OCR使用有無で共有URLのキー構造・エンコード仕様差分なし。
- 判定: 互換維持。

### 3.4 ランキングpayload互換
- 結果: OCR使用有無でランキングpayload構造差分なし。
- 判定: 互換維持。

### 3.5 未確認OCR値の比較計算混入防止
- 結果: A/Bいずれかに未確認・未確定項目が残るケースでは比較実行がブロックされ、比較計算への混入なし。
- 判定: 要件どおり。

### 3.6 A/B取り違え誤反映防止
- 結果: OCR反映先は開始時指定の系統（AまたはB）のみに限定され、反対系統への誤反映は0件。
- 判定: 要件どおり。

### 3.7 低信頼度候補の自動確定防止
- 結果: 低信頼度候補は未確定扱いを維持し、自動確定は0件。
- 判定: 要件どおり。

### 3.8 OCR失敗時の対象系統のみ手動fallback
- 結果: OCR失敗時は失敗した系統のみ手動入力へ遷移し、反対系統の入力値は保持。
- 判定: 要件どおり。

## 4. 限定導入判定

- 判定: **条件付き継続**
- 根拠:
  1. 互換性（保存payload・共有URL・ランキングpayload）および比較結果正当性は維持。
  2. A/B取り違え誤反映率は0.0%を維持。
  3. 低性能端末p95（4.2秒）はロールバック閾値近傍のため継続監視が必要。

## 5. 第2サイクル進行判断

- 判断: **第2サイクル計測へ進行可（改善Issueは同時起票推奨）**
- 理由:
  1. 重大な互換性破壊・誤反映インシデントは未検知。
  2. fallback率/補正率/離脱率は許容範囲だが、低性能端末p95改善余地が残る。
  3. したがって導入停止ではなく、監視条件強化の上で第2サイクル計測を継続する。
- 改善Issue候補:
  - 低性能端末カテゴリのOCR処理時間p95短縮
  - fallback発生要因の内訳可視化
  - 比較前確認離脱要因の文言/導線改善検証（仕様変更なしの範囲）

## 6. security / privacy / license / release / support 観点

- security: OCRメタは永続payloadへ保存せず、UI一時状態のみで扱う運用を継続。
- privacy: 評価画像はRepository非保存。個人情報・アカウント情報を含む画像は不使用。
- license: 非公式ファンツールとして、公式素材利用可否は引き続き「要確認」。
- release: `/compare` OCR限定導入は条件付き継続。
- support: OCR失敗時の対象系統のみ手動fallbackで比較フロー継続可能。
