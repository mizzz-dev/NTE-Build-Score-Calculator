# Issue #107 作業ログ（/card OCR入力補助 最小実装）

- 日付: 2026-05-14
- 担当: Codex
- 対象Issue: #107

## 1. 変更してよい層 / 禁止領域
- 変更可:
  - `/card` 入力UI導線（OCR補助の一時状態）
  - 既存OCR補助Panelの再利用拡張（画面タイトル差し替え）
  - payload非混入を担保するユニットテスト
  - ステータス/Issueドキュメント更新
- 禁止領域:
  - `/compare` 実装
  - 保存payload仕様の変更
  - 共有URL仕様の変更
  - ランキングpayload仕様の変更
  - DB migration / auth / infra / deployment設定

## 2. 実施内容
1. `/card` 画面に OCR入力補助導線を追加。
2. OCR結果を下書き状態で保持し、フォーム反映後も「生成前確認チェック」を必須化。
3. 低信頼度項目は既存OCRロジックに従い自動確定しない運用を維持。
4. OCR失敗時は手動入力へ戻るfallback文言を追加。
5. 保存payload/ランキングpayloadにOCRメタ情報が混入しない検証テストを追加。

## 3. KPI記録方針（/card限定公開）
- OCR処理時間 p95（全体/端末カテゴリ別）
- 未確定項目残存率（生成前確認時点）
- fallback率（OCR失敗・手動入力回帰）
- 反映前確認離脱率

## 4. ADR要否
- 今回はADR不要。
- 理由: 既存設計（OCRは一時状態・payload非混入）内での最小導線追加であり、恒久アーキテクチャ変更を伴わないため。
