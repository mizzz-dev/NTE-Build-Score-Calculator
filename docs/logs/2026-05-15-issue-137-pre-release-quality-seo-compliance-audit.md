# Issue #137 作業ログ（正式リリース前 品質・SEO・規約表示確認）

- 作成日: 2026-05-15
- 対象Issue: #137
- 対象導線: `/score` `/card` `/compare`（OCR使用/未使用）
- 方針: 非公式ファンツール前提を維持し、法務判断は確定せず「要確認」として整理する

## 1. 実施概要

正式リリース前の品質・SEO・規約表示・権利確認観点を、既存実装と既存Runbookに照らして棚卸しした。

- `/score` `/card` `/compare` の主要導線とOCR補助導線を確認。
- OCR未使用時の手動入力継続導線を確認。
- OCR使用時に保存payload・共有URL・ランキングpayloadへOCRメタ情報を混入させない実装・テストを確認。
- 低信頼度候補を自動確定しない方針を確認。
- 非公式表記、誤認表示、SEO metadata、規約/免責/プライバシー/問い合わせ導線の現状を確認。
- secret取扱いと `.env.local` 未コミットを確認。

## 2. 確認結果（Issue要件1〜13）

### 2.1 主要導線（1,2,4）

1. `/score` `/card` `/compare` は各ページコンテナが有効で、手動入力導線が主導線として維持されている。  
2. OCRは補助パネルとして併設され、失敗時は再試行または手動入力継続が明示されている。  
3. OCR候補は未解決項目が残る場合に反映不可で、確認済みチェックを通す設計が維持されている。  

判定: **OK（実装方針維持）**

### 2.2 payload互換・OCRメタ非混入（3）

1. payload builderのテストで、`confidence` `matchType` `rawText` `candidates` 等が保存/ランキングpayloadに含まれないことを検証済み。  
2. `/card` のUI文言でも「保存payload・共有URL・ランキングpayloadにはOCRメタを保存しない」旨が明示されている。  

判定: **OK（仕様維持）**

### 2.3 誤認表示・非公式表記（5,7）

1. ヘッダーとフッターに「非公式ファンツール」表記が存在。  
2. フッターに「公式素材の利用可否: 要確認」を表示。  

判定: **概ねOK（最低限の明示あり）**

### 2.4 SEO/metadata/OGP整合（6,9）

1. `layout.tsx` metadataは `title` / `description` のみで、OGP/Twitter/robots/canonical等の公開前最終整備は未実施。  
2. descriptionが「開発基盤」であり、正式公開時の機能説明としては検索流入向け情報量が不足。  

判定: **要改善（公開前P1）**

### 2.5 規約・免責・プライバシー・サポート（10,11）

1. `/contact` は空状態ページで、問い合わせ導線は存在するが運用定義は未確定。  
2. 利用規約/免責/プライバシー方針の独立ページは未実装（少なくとも `src/app` 配下に該当routeなし）。  
3. OCRパネル上で「画像はブラウザ内処理のみ」「サーバー保存しない」「OCRは一時ドラフト」が明示されている。  
4. OCR結果の最終確認責任（手動確認必須）の文言は複数導線で維持。  

判定: **要改善（公開前P0〜P1）**

### 2.6 secret / env確認（12,13）

1. Supabase設定は `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` のみ参照。  
2. trackedなenvファイルは `.env.example` のみで、`.env.local` はコミットされていない。  
3. service role key参照実装は確認されない。  

判定: **OK（現状問題なし）**

## 3. 公式素材・名称・画像・ゲーム内情報の扱い（8）

法務判断は本Issue対象外のため確定せず、以下を「要確認」とする。

1. 「NTE / Neverness to Everness」名称利用のガイドライン適合性。  
2. スクリーンショット利用・OCR入力対象画像の権利整理（ユーザー投稿画像を含む）。  
3. 公式ロゴ/素材の将来利用可否（現状は未使用・要確認方針）。  
4. 公開時の免責文言（正確性・非保証・責任範囲）の法務レビュー要否。
