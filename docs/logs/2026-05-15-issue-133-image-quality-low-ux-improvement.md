# Issue #133 作業ログ（image_quality_low 起因の事前撮影ガイド・再撮影導線改善）

- 作成日: 2026-05-15
- 対象Issue: #133
- 関連Issue/PR: #127, #131, #133

## 1. 実施概要

`/score` `/card` `/compare` で共通利用しているOCR入力補助パネルの案内文を最小差分で更新し、OCR実行前の撮影品質ガイドと、OCR失敗時の再撮影/手動入力の選択導線を明確化した。

## 2. 変更内容

1. OCR実行前チェック文言を更新（共通）
   - 文字が読める明るさ
   - 傾き・ブレ回避
   - 対象ステータス部分の欠け防止
   - 個人情報/アカウント情報の写り込み防止
2. OCR失敗時エリアに、再撮影での再試行と手動入力継続の両導線を明記。
3. `/compare` と `/card` の補足エラー文言に再撮影導線を追加。
4. OCRアルゴリズム・OCR信頼度計算・保存payload仕様は未変更。

## 3. 影響確認

- `/score`: 既存入力値保持の案内は維持。
- `/card`: OCR失敗時に再撮影または手動継続を選べる文言へ更新。
- `/compare`: A/B片側失敗時でも再撮影か手動継続を選べる文言へ更新。
- 低信頼度候補の自動確定は引き続き未実施。

## 4. 非変更宣言
