# 要件定義書

## 1. ドキュメント目的
本書は、NTE Build Score Calculator のウォーターフォール型開発に向けて、実装前に機能要件・非機能要件・制約・未確定事項を明確化する。

## 2. 開発方針
- 開発方式: ウォーターフォール
- 方針: 全機能を対象に要件→設計→実装の順に進行
- ドキュメント・PR・レビュー言語: 日本語統一

## 3. 機能要件（ユーザー向け）
1. トップページ
2. スコア計算ページ
3. 装備1個単位のスコア計算
4. キャラ全体ビルドスコア計算
5. ロール別評価 / キャラ別評価
6. ビルド比較
7. ビルドカード生成
8. 画像保存（形式選択可能）
9. SNS共有
10. URL共有
11. 共有URLから入力復元
12. プリセット一覧
13. ステータス一覧
14. 使い方
15. FAQ
16. アップデート情報
17. お問い合わせ / フィードバック
18. Googleログイン
19. ゲスト利用
20. ユーザー別ビルド保存
21. ゲスト履歴保存
22. ランキング
23. OCR / スクショ読み取り
24. ダメージシミュレーター
25. 多言語対応（日本語・英語・韓国語）

## 4. 機能要件（管理者向け）
1. 管理画面
2. キャラ管理
3. ロール管理
4. ステータス管理
5. スコア重み管理
6. プリセット管理
7. お知らせ管理
8. FAQ管理
9. アップデート履歴管理

## 5. 非機能要件
- パフォーマンス: 一般的なモバイル通信環境でストレスなく利用可能
- 可用性: 公開Webサービスとして安定運用
- セキュリティ: 認証・認可・入力バリデーション・監査ログを考慮
- SEO: 主要ページに対して対応
- OGP: 共有時の視認性確保
- アクセシビリティ: 主要導線で配慮
- 監視: アクセス解析（GA）・エラー監視（Sentry等）

## 6. 対応ブラウザ
- Chrome
- Safari
- Edge
- Firefox
- iOS Safari
- Android Chrome

## 7. リリース完了条件
- 定義された初期対象機能が利用可能
- 重大バグが解消されている
- 必須テストを完了している
- 運用ドキュメント（利用規約/プライバシーポリシー/リリースチェック）を整備

## 8. テスト方針
- 単体テスト
- 計算ロジックテスト
- E2Eテスト
- レスポンシブ確認
- 本番ビルド確認

## 9. 制約事項
- 初回PRでは実装コードを追加しない
- 要件定義と開発ルールの整理を優先

## 10. 未確定・要確認事項
- NTE公式画像・ロゴ・キャラ画像の利用可否
- ファンツール運用の法的・規約上の範囲
- 正式な装備名称・ステータス名称
- Cartridge / Module / Gear / Console の厳密仕様
- ステータスの最小値 / 最大値 / 小数点桁数
- セット効果・装備効果の算入ルール
- キャラ別最適ステータス
- ダメージ計算式
- OCR対象スクリーンショットの仕様

## 11. Issue #3 反映（スコア仕様詳細化）
- スコア計算は「装備単位 -> ビルド合算 -> ランク判定」の3段階で定義する。
- 評価プロファイルの優先順位は「キャラ別 > ロール別 > デフォルト」とする。
- ランクは 10 段階（D〜SS）を採用し、閾値は設定データとして管理可能とする。
- カスタム重み付けはユーザー保存可能とし、標準プロファイルとの差分適用方式を採用する。
- 未確定ゲーム仕様は「要確認」として分離し、仮実装を禁止する。
